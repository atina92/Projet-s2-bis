Comment est construit le Web ?

A 35 ans il développe alors The WWW ( The World Wide Web)
Il met en place à l'aide de son associé Robert Cailliau l'URL pour donner une addresse aux pages, le langage HTML pour les afficher, et le protocole HTTP pour les lier entre elles.
  • L'URL

    • Une URL (Uniform Resource Locator) est un format de nommage universel pour désigner une ressource sur Internet. Il s'agit d'une chaîne de caractères ASCII qui se décompose en cinq parties :

    1. Le nom du protocole (le plus largement utilisé est le protocole HTTP)
    2. Identifiant et mot de passe ( facultatif et visible dans l'URL)
    3. Le nom du serveur
    4. Le numéro de port (falcultafif dans le cas où le port par défaut est le port numéro 80)
    5. Le chemin d'accès à la ressource

  • Le protocole HTTP

    • HTTP (= Hypertext Transfer Protocol) est le protocole qui permet d’échanger des pages web entre le client et le serveur. HTTPS est identique à HTTP, avec le "S" en plus qui signifie "Secure". Les pages sont cryptées pour garantir la confidentialité des données échangées (données personnelles, financières etc...). Il combine HTTP à SSL (= Secure Socket Layer), une couche (= layer) supplémentaire sécurisée. Ce protocole va créer une sorte de canal sécurisé entre le client et le serveur.

  • Le language HTML

    • HTML (= Hypertext Markup Language) est le langage de balisage standard pour les documents conçus pour être affichés dans un navigateur web. Il peut être assisté par des technologies telles que les feuilles de style en cascade (CSS) et les langages de script tels que JavaScript

Le nom initial du web dans son mémo était The Information Mesh (=le maillage d'information) mais l'acronyme aurait été TIM, son prénom, ce qu'il trouve "trop narcissique".

Malgré ses multiples récompense, il déclare n'avoir rien inventé:

"J'ai juste emprunté un certain nombre de choses qui existaient déja.(...)
Construire le Web, je ne l'ai pas fait tout seul."
 Tim Beerners-Lee, 2004


Le CERN diffuse le code source du web en 1993 et en un an, le nombre de serveur web se multiplie! Tous utilisent le protocole HTTP, une toile géante se tisse sans que Tim Beerners-Lee n'en tire profit.
Il rejoint le MIT (Massachusetts Institute of Technology) en 1994 et fonde le World WIde Web Consortium (W3C). Cela a permit d'obtenir les standards d'un web ouvert, libre et transparent En effet pour lui le Web permet le partage de connaissance...
Malheuresement, le web n'est plus utilisé que dans cette optique.

Qu'est-ce qu'est le web aujourd'hui ?


Tim Beerners-Lee a souhaité garder son invention public car . Mais 30 ans après sa création, le web semble avoir échappé à son fondateur...
En effet ce génie énonce en 2012 que l'une des menaces les plus importantes auquels peut faire face le web est lorsqu'une personne, un gouvernement cherche à prendre un contôle sur l'ensemble ou une grande partie d'une création, en bloquant ou en espionnant les utilisateurs, voir même en censurant. Le gouvenement peut vous voir comme une menace parce que vous consultez certains sites et vous mettre en prison. Il confirme donc que garder le web ouvert est très important.
En 2018, dévasté, par la manipulation des données personnelles, il créé Inrupt un coffre fort pour permettre à chacun de stocker ses données
